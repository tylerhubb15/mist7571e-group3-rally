My role on the project was Code Quality. Throughout the project, I reviewed the codebase to make sure our application met the project requirements and the code quality standards outlined in our project guidelines. I checked things like coding conventions, reusable components, loading and error states, secure API key handling, and documented any issues I found and shared them with the developers. One thing that didn't go as planned was finding a few files that were still using loose equality operators instead of strict equality, which didn't meet our coding standards. I documented those locations, the developers made the necessary changes, and after the updates were merged into the main branch, I reran the review to verify the issue had been resolved. It was a good example of how the quality assurance process helps catch small issues before the final submission.
